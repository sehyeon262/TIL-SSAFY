# 💡 Vue에 대한 이해

## Vue 란?
- JavaScript를 훨씬 쉽고 편하게 쓰게 해주는 프레임워크(도구)

- JS만 쓰면 힘든 것들을 Vue가 자동으로 대신 해줌.


## 현실 비유로 이해하기
### 🔨 JavaScript = 맨손으로 집 짓기
- 못 직접 박아야 함

- 벽도 직접 쌓아야 함

- 문도 직접 만들고 달아야 함

=> 가능하지만 시간이 오래 걸리고 코드 복잡함.


### 🏗️ Vue = 이미 다 준비된 “건축 세트”

- 문, 창문, 벽 템플릿 제공

- 자동으로 연결되는 구조 제공

- 데이터 바꾸면 화면 자동 업데이트

- 이벤트 처리도 더 간단하게 가능

=> 건물 구조(앱)를 훨씬 빠르고 안정적으로 만들 수 있음


## Vue가 왜 필요해?
- JavaScript로 큰 웹앱을 만들면 ↓

    - 코드가 더러워짐

    - 개수 많으면 html 조작 어려움

    - 데이터가 바뀌면 화면을 일일이 다시 고침

    => Vue는 이런 문제를 해결함.



# ✏️ 추가 개념

## 1. Page = HTML + CSS + JS

### 🏠 HTML = 방의 구조

- 문이 있음

- 창문이 있음

- 책상이 있음


### 🎨 CSS = 그 방의 인테리어

- 벽 색깔 바꿈

- 글자 크기 바꿈

- 위치 바꿈


### 🤖 JavaScript = 자동문, 불 켜지고 끄는 시스템, 움직이는 장치

- 버튼 누르면 문이 자동으로 열리게 함

- 알림창 띄움

- 클릭하면 글자 바뀌게 함

- 스크롤하면 메뉴가 따라다니게 함

=> “인터랙션(사용자와 상호작용)”을 만드는 애가 JS



## 2. AJAX

### 정의 (Asynchronous JavaScript And XML)
- **페이지 전체를 새로고침하지 않고, 일부분만 업데이트할 수 있게 해주는 기술**


### 현실 비유로 이해하기

#### 🔵 일반 웹 방식 (새로고침)
- 피자를 시켰다고 치자.

- “펩시 하나 추가요!” 라고 말하면
    
    => 가게가 피자부터 다시 굽고, 처음부터 전체 주문을 다시 포장해서 보내주는 느낌 (전체 새로고침)


#### 🟢 AJAX 방식 (부분 업데이트)
- “펩시 하나만 추가요!”라고 하면
    
    => 가게가 **펩시만 따로** 보내주는 느낌 (필요한 정보만 받아오기)

=> 기존 페이지는 그대로 두고 필요한 데이터만 서버에서 가져오는 기술!

### 예시로 이해하기 (진짜 핵심)
1) 새로고침 방식

    - 버튼 클릭 → 페이지 전체가 새로고침 → 결과가 등장

2) AJAX 방식

    - 버튼 클릭 → 서버에 요청만 보냄 → “답변 데이터”만 받아와서
→ 페이지 일부에 JS로 딱! 표시
